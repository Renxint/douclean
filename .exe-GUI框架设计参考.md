# 这是一份思路框架文档，仅作分析用。实际开发时请以项目需求为准。

---

一个专业的 Windows .exe 桌面工具，从**用户第一次启动到日常使用**的完整生命周期，GUI 框架应该覆盖以下结构：

---

## 1. 启动与生命周期

### 1.1 单实例检测
- **为什么需要**：用户双击 exe 两次 → 启动两个进程 → 资源冲突（cookie文件、端口、配置文件写入竞态）
- **实现方案**：`QSharedMemory`（PyQt）、Windows 命名 Mutex、或本地 socket
- **体验要求**：检测到已有实例时，激活已有窗口并退出新进程，而不是冷冰冰弹个"已运行"错误

### 1.2 启动参数
- 支持命令行参数：`抖净.exe --silent`（静默启动到托盘）、`抖净.exe "https://v.douyin.com/xxx"`（直接解析链接）
- 结合单实例检测：新进程将参数传递给已有进程，由已有进程处理

### 1.3 崩溃恢复
- 全局异常捕获 `sys.excepthook`，记录到 `crash.log`
- 崩溃时给出友好提示，而非 Windows 默认的"程序已停止工作"
- 可选：看门狗进程自动重启

### 1.4 窗口状态持久化
- 记住上次关闭时的窗口位置、大小、是否最大化
- 下次启动恢复

---

## 2. 窗口管理

### 2.1 主窗口
- **最小尺寸**：防止用户缩到看不见
- **标题栏**：应用名 + 版本 + 文档标题（如有）
- **任务栏图标**：与托盘图标共用，`app.ico`
- **关闭行为**：点 X → 最小化到托盘（消费者工具的标准行为），而非直接退出

### 2.2 页面导航
- `QStackedWidget` 多页面切换（首页 → 功能页 → 设置页）
- 返回按钮 + 面包屑（可选）
- 页面切换动画（可选，`QPropertyAnimation`）

### 2.3 状态栏 (`QStatusBar`)
- 永久显示：版本号、托盘提示
- 临时显示：下载进度、操作结果、错误信息（超时自动消失）
- 右侧可选：网络状态、Cookie 状态

---

## 3. 系统托盘

### 3.1 托盘图标
- 应用 .ico 同款
- ToolTip 显示应用名 + 当前状态（"抖净 · 就绪" / "抖净 · 下载中"）

### 3.2 托盘右键菜单
标准结构：
```
显示主窗口          ← 双击托盘图标同效果
──────────────
开机自启    ✓       ← Checkable，读/写注册表
──────────────
关于 抖净           ← 版本+作者+链接
检查更新            ← 手动触发
──────────────
退出                ← 真正退出，停止所有任务
```

### 3.3 托盘通知 (`showMessage`)
- 下载完成、更新可用、错误提醒
- 通知应该有节制——不要每个小操作都弹

---

## 4. 设置系统

### 4.1 存储方案
- 轻量级：JSON 文件（`settings.json`），与 exe 同目录
- 需要多层时用 SQLite
- 存储内容：字体、窗口几何、下载路径、Cookie过期时间、开机自启状态、主题偏好

### 4.2 设置分类
```
外观：字体、字号、主题色
下载：默认保存路径、并发数、文件命名规则
Cookie：查看当前状态、粘贴更新、过期时间
高级：开机自启、通知开关、代理设置
```

### 4.3 设置页 UI
- 独立页面或对话框
- 即时预览（字体、主题色修改立刻看到效果）
- "恢复默认"按钮

---

## 5. 更新系统

### 5.1 版本检查
- 启动时后台线程静默检查（不阻塞 UI）
- 检查 URL 可配置，返回 JSON：`{"version": "1.0.1", "url": "...", "note": "..."}`

### 5.2 更新提示
- 有新版本 → 非阻塞弹窗（不在最前、可忽略）
- 显示：当前版本 / 最新版本 / 更新内容 / 下载按钮
- "忽略此版本"选项

### 5.3 下载更新
- 方案1：打开浏览器下载（简单，推荐小工具）
- 方案2：内置下载器下载 zip → 提示手动替换
- 方案3：自动替换 exe（复杂，需要启动器/脚本）

---

## 6. 通知系统

### 6.1 分级
| 级别 | 场景 | 方式 |
|------|------|------|
| 成功 | 下载完成 | 托盘气泡 + 状态栏 |
| 信息 | 最小化到托盘 | 托盘气泡（2秒消失） |
| 警告 | Cookie 即将过期 | 状态栏黄字 + 可选托盘 |
| 错误 | 下载失败 | 日志区红字 + 状态栏 |

### 6.2 节制原则
- 通知不是越多越好——用户会麻木
- 长时间任务（批量下载）只在全部完成时通知一次
- 别在通知里堆技术细节，用户只看结果

## 8. 主题与视觉

### 8.1 暗色/亮色
- 当前：Cinema Dark 暗色主题
- 未来可选：亮色模式切换

### 8.2 设计系统
- **颜色系统**：主背景 / 卡片背景 / 边框 / 主强调色 / 文字（主/辅/弱） / 语义色（成功绿/警告黄/错误红）
- **间距系统**：统一 4px 倍数（8/12/16/24/32）
- **圆角系统**：控件 8px / 卡片 16px / 按钮 8px
- **阴影系统**（可选，QGraphicsDropShadowEffect）

### 8.3 字体
- 等宽字体：日志区
- 系统默认字体：正文
- 字体大小可调（已有）

### 8.4 图标
- 功能图标用文字/数字（免去图标库依赖）
- 语义色点（绿/黄/红）表示状态
- 应用图标 .ico 需多分辨率（16-256px）

### 9.1 快捷键

### 9.2 Tab 序

- `setTabOrder()` 或在布局中自然排列

## 10. 日志与调试

### 10.1 用户可见日志
- `QTextEdit` 只读，显示下载进度
- 颜色区分：`[OK]` 绿、`[ERROR]` 红、`[>>]` 灰

### 10.2 开发者日志
- `EXE_DIR/_debug.log`，仅开发模式下写入
- 打包后默认关闭，通过设置或命令行 `--debug` 开启

### 10.3 日志轮转
- 单文件，超过 1MB 自动截断保留后半
- 或保留最近 7 天

---

## 11. 错误处理框架

### 11.1 全局异常捕获
```python
def global_exception_handler(exc_type, exc_value, exc_tb):
    # 写 crash.log
    # 弹 QMessageBox 友好提示
    # 不直接 sys.exit()
sys.excepthook = global_exception_handler
```

### 11.2 线程异常
- QThread 内捕获所有异常，通过信号传回主线程
- 不静默失败——至少日志里要有

### 11.3 网络异常
- 超时 / DNS 失败 / 404 → 用户友好提示，而非 dump requests 异常堆栈

## 12. 文件关联与拖放（可选但加分）

### 12.1 拖放
- 从浏览器拖链接到窗口 → 自动填入输入框
- `setAcceptDrops(True)` + `dragEnterEvent` / `dropEvent`

### 12.2 命令行传参
- `抖净.exe "https://v.douyin.com/xxx"` → 启动后自动跳转单视频页并填入链接
- 结合单实例检测，传参给已有实例

## 13. 多语言 / 国际化

### 13.1 当前状态
- Qt 自带的对话框已加载 `qtbase_zh_CN.qm` 中文翻译
- 界面文字硬编码中文

### 13.2 如果要扩展
- 所有界面文字提取到 `strings_zh.json` / `strings_en.json`
- `QTranslator` 加载对应 qm 文件
- 设置页加语言切换（需重启生效）

---

## 14. 安全与权限

### 14.1 Cookie 存储
- 明文存 JSON → 最低安全要求：至少放在与 exe 同目录的 `data/` 下
- 不要上传 Cookie 到任何服务器（反馈系统应过滤 Cookie 字段）

### 14.2 联网最小化
- 仅在需要时联网（下载视频、检查更新、发送反馈）
- 其他时间无后台网络请求
